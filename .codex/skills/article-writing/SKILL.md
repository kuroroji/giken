# 記事作成の反復手順

対象: `src/content/articles/*.md`。文体・表記・太字・表現安全の常時ルールは `AGENTS.md` に従う。

## 0. 原則: いきなり本文を書かない

ユーザー(くろろじ本人)を「テーマに詳しい一次情報の提供者」として扱い、相談フェーズを先に行う。
目的は、一般情報を並べた記事ではなく**本人にしか書けない記事**を作ること。
ユーザーが「相談なしで書いて」と明示した場合のみ省略してよい。

## 1. 相談フェーズ(インタビュー)

- 質問は一度に**1〜3個だけ**。番号付きリストの尋問ではなく、悩み相談を聞く相手のような会話調で聞く
- 回答を受けたら: 重要な主張を短く整理 → 使える具体例を抽出 → 曖昧な点を特定 → 次の質問1〜3個
- 仮説や選択肢を提示して答えやすくする(「AとBならBが核だと思っていますが合ってますか?」)
- 優先順位: ①記事の結論・きっかけの違和感 → ②本人の実体験・失敗・迷い → ③判断基準 → ④言い回し・表現の温度感
- 終了条件: 中心主張・想定読者・本人の具体例(1つ以上)・一般論との差分・構成、が揃ったら「記事の核は固まりました」と宣言し、仮タイトル・構成・各見出しに入れる本人の経験を提示する
- 執筆時: 本人が話していない経験を捏造しない。推測を事実として書かない。本人の語彙と温度感を残す

## 2. frontmatter 設計

スキーマは `src/content.config.ts`。主な項目:

```yaml
title: 〜とは?〜します!    # 半角!?、「|」禁止、120字以内
description: 〜            # 200字以内。検索結果に出る説明文
publishedAt: YYYY-MM-DD
category: game | youtube-guide | promotion   # src/data/categories.ts
tags: [タグ1, タグ2]
eyecatchLines:             # 自動生成アイキャッチ/OGPの改行位置
  - 1行目
  - 2行目
featured: true             # トップの注目記事に出す場合
contactForm: true          # 記事本文直後に問い合わせフォームを埋め込む(案件導線記事のみ)
faq:                       # 設定すると折りたたみFAQ+FAQPage構造化データ+目次項目が自動で付く
  - q: 質問?
    a: 回答。
```

- FAQは案件系・ハウツー系記事で有効。10問前後。回答はAGENTS.mdの表現安全ルールに従う
- `draft: true` で下書き(本番ビルドから除外)

## 3. 記事の構成パターン

- リード: 読者の悩みを代弁 → 自己紹介(僕は〜する立場です)→ 前提の明示 → 「先に結論を言うと〜」
- 営業系記事はリード直後に「この記事の結論」の引用ボックス(blockquote+箇条書き5点程度)を置く
- リードの自己紹介直後にYouTubeチャンネルボタンを置く:

```html
<div class="store-links">
  <a class="store-btn store-btn--youtube" href="https://www.youtube.com/@KUROROJI" target="_blank" rel="noopener">▶ くろろじちゃんねるを見てみる</a>
</div>
```

- 大見出し(##)は口語の問いかけ+答えの形。目次(h2のみ表示)だけ読んで流れが分かること
- 読者がそのまま行動できる素材(メールテンプレ等)はコードブロックで入れる(コピーボタンが自動で付く)
- 内部リンク: `/promotion/`(案件詳細・実績)、`/profile/`(プロフィール)。実在ページのみ

## 4. 画像

- 置き場所: `public/images/articles/`(記事内参照は `/images/articles/ファイル名`)
- ゲームのキービジュアルは `.webp`、UIスクリーンショットは `.png` でよい
- ユーザーがチャットに貼った画像は、直後ならクリップボードから保存できる:
  `powershell.exe -sta -noprofile -command '$img = Get-Clipboard -Format Image; if ($img) { $img.Save("保存先.png") }'`
- alt テキストは「何のどの部分か」まで書く

## 5. CTA(才流メソッド準拠)

- 記事前半: テキストCTA(ページ側で自動表示。個別記事では不要)
- 記事末尾: ページ側のリッチCTA(From Kuroroji)が自動で付く
- 案件導線記事は `contactForm: true` で本文直後にフォームを埋め込み、本文末尾を「まずは相談だけでも大丈夫です」トーンで締める(圧を出さない。無理に受けないことも明記)
- ストアボタンの色分けは game-article スキルを参照

## 6. ローカル検証

1. 開発サーバー起動(`npm run dev`、ポート4321)し、`http://localhost:4321/articles/<slug>/` を開く
2. 本文テキストに生の `**` が残っていないか確認(残っていたらAGENTS.mdの太字の罠を参照して修正)
3. 目次・FAQ・フォーム・画像・動画埋め込み・ボタンの表示確認
4. `npm run build` が全ページ成功すること

## 7. 公開

1. ユーザーに記事を通し読みしてもらい、修正要望を反映(見出し・太字の増減・言い回しの往復が通常発生する)
2. 公開の明示的なOKをもらってから commit → `git push origin main`
3. デプロイ後、`curl` で本番URLに新しい文言が含まれることを確認して報告
